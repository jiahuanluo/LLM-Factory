## 一、为什么要做:NLU 已被验证更优,我们把底座建好了

### ① 价值已验证:NLU 模型在多个场景优于当前 ML 模型 ⭐

- 在 **风险类模型**(违约预测、还款意愿等)与 **经营类模型**(收入预测、久期等)多个场景,经实测对比:
- **NLU 模型效果优于"当前特征下的机器学习模型"**
- 说明部门沉淀的海量业务文本(征信 / 合同 / 发票 / 税务 / 财报)确实产生了**增量价值**
- **结论**:NLU 是风控建模的下一级,**方向已被业务验证,不是概念炒作**

### ② 但有瓶颈:从零训 NLU,时间与能力都扛不住

- **能力差异大**:NLU 训练涉及 **预训练底座、tokenization、fine-tuning、分布式训练** 等专业知识,与 ML 模型所需能力差异显著
- **训练周期长**:从预训练开始训,周期是 **月级甚至季度级**,业务侧等不起
- **模型科室搞不定**:模型科室同学熟悉结构化建模,但 **没训过 NLU**,只能找算法排期
- 现状:想做 NLU → 找算法排期 → 流程长、响应慢

### ③ 我们已经准备好了:5 大底座 + 自助微调平台

- 经过 **历次模型迭代积累**,**5 大金融 NLU 底座已训好**(主模型 / 企业 / 个人征信 / 税银 / 发票)
- 在底座基础上做下游微调,**周级交付** NLU 模型
- 工作流闭环:**选底座 → 微调 → 跟 ML 对比 → 选性能最优模型**
- 让模型科室同学能像"训 ML 模型"一样"训 NLU 模型"

> 💡 **一句话**:NLU 已被验证更优,我们把底座和平台建好了 —— 业务侧可以快速跑出 NLU 模型,与现有 ML 对比择优,让每个业务都能用上"更优的那一个"。

## 二、项目定位:风险部 NLU 自助训练平台

> **不是新算法,是"自助训练平台"——模型科室同学带着数据来,带着上线模型走。**

把 **"业务文本 → 私域底座 → 下游微调 → 上线"** 这条链路 **标准化、自动化、自助化**:

```
                    模型科室同学(使用方)
                          │
                   带业务数据来
                          ▼
   ┌──────────────────────────────────────────┐
   │            LLM-Factory 自助平台            │
   │                                          │
   │   ① 5 大私域底座(选一个)                │
   │      主模型 / 企业 / 个人征信 / 税银 / 发票│
   │                                          │
   │   ② 一键 YAML 配置微调训练               │
   │      (不用写代码,改配置就能训)         │
   │                                          │
   │   ③ 自动评估 + 上线                       │
   │      (内置 AUC / KS / MAE 等指标)        │
   └──────────────────────────────────────────┘
                          │
                          ▼
                  业务专属 NLU 模型
                  (违约 / 收入 / 久期 / 还款意愿 ...)
```

**职责分工**:
- **算法侧(我们)**:维护 5 大底座、迭代平台、提供培训和最佳实践
- **模型科室侧(使用方)**:基于底座 + 业务数据,自助训练模型

## 三、⭐ 核心战略资产:5 大私域预训练底座矩阵

> ✅ 金融版"NLU 底座矩阵",**全部已训练完成,模型科室同学可直接选一个做下游微调**。

| 底座模型 | 主要语料 | 适用下游任务 |
|---|---|---|
| **主模型预训练** | 部门全量业务文本(综合底座) | 通用 NLU 任务、跨业务基线 |
| **企业侧预训练** | 工商 / 财报 / 企业征信文本 | 企业违约预测、企业授信、对公风控 |
| **个人征信预训练** | 个人征信报告 / 信用文本 | 个人违约预测、消费贷风控、客户分层 |
| **税银预训练** | 税务数据文本化 / 涉税文本 | 收入预测、纳税信用评估、税务反欺诈 |
| **发票预训练** | 发票文本 / 票据 OCR 文本 | 发票风控、虚假发票识别、交易真实性核验 |

**模型科室同学使用方式**:**根据任务场景选底座 → 拿业务数据微调 → 上线**
- 例:做"企业违约预测" → 选 **企业侧底座** → 灌业务标注数据微调 → 上线
- 例:做"个人还款意愿预测" → 选 **个人征信底座** → 灌历史催收数据微调 → 上线

**战略价值**:
- 沉淀 **部门不可复制的模型资产**(竞对拿不到这些语料)
- 让模型科室同学 **站在底座肩膀上**,无需从零训 NLU,冷启动 **天级交付**
- 形成 **数据飞轮**:用得越多 → 底座越强 → 下游效果越好

## 四、四大业务任务(模型科室已用平台跑出上线模型)

> ✅ 不讲技术栈,讲业务能拿 NLU 解决什么问题。

### 任务一:违约客户预测(信贷风控核心)

- **业务问题**:提前识别可能违约的对公 / 零售客户,降低不良率
- **NLU 方案**:模型科室基于 **企业侧 / 个人征信底座** 微调二分类模型,把征信/工商/合同文本信息用起来
- **业务指标**:**AUC / KS**(已内置,直接对标监管上线标准)
- **业务收益**:不良率 ↓、通过率 ↑、拨备成本 ↓

### 任务二:收入预测(授信 / 额度核心)

- **业务问题**:客户真实收入难获取,影响授信额度与定价
- **NLU 方案**:模型科室基于 **税银底座 / 个人征信底座** 微调回归模型,从税务 / 征信文本预测收入
- **业务指标**:MAE / RMSE
- **业务收益**:授信精度 ↑、定价更准、风险定价能力 ↑

### 任务三:久期预测(资产负债管理)

- **业务问题**:贷款 / 资产久期影响利率风险与流动性管理
- **NLU 方案**:基于主模型微调回归模型,从客户与合同文本预测久期
- **业务收益**:利率风险敞口 ↓、资产负债匹配 ↑

### 任务四:还款意愿预测(贷后 / 催收)

- **业务问题**:识别"有能力但不愿还"的客户,精准催收
- **NLU 方案**:模型科室基于 **个人征信底座** 微调分类 / 回归模型,区分还款意愿强度
- **业务收益**:催收回收率 ↑、催收成本 ↓、客诉率 ↓
